# Web Development — Learning

Curated. Not exhaustive — just resources that are actually worth your time, organized by where you are in the learning curve.

## Reference (read these first, return to them often)

- **[MDN Web Docs](https://developer.mozilla.org)** — *the* reference. HTML, CSS, JS, Web APIs. The single resource you should bookmark and habit your way back to weekly.
- **[web.dev](https://web.dev)** — Google's developer guides. Best on performance, accessibility, and modern platform features.
- **[caniuse.com](https://caniuse.com)** — browser support tables. Check before adopting any feature.
- **[The TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/intro.html)** — official, well-written, complete in one weekend.
- **[The React Docs (react.dev)](https://react.dev)** — the rewrite from 2023 is the best framework documentation in any ecosystem. Work through "Learn React" cover to cover.

## Beginner — building the platform foundation

- **[Josh Comeau — *The Joy of React*](https://www.joyofreact.com)** — paid, expensive, worth it if React is the path. The single best React course.
- **[Josh Comeau — CSS for JavaScript Developers](https://css-for-js.dev)** — paid, also worth it. Builds the *intuition* most CSS courses skip.
- **[Wes Bos — JavaScript30](https://javascript30.com)** — free, 30 vanilla-JS exercises, no framework. Excellent platform fluency builder.
- **[The Odin Project](https://www.theodinproject.com)** — free, fundamentals-first curriculum. Slower than alternatives, more thorough.
- **[Frontend Masters — Bootcamp](https://frontendmasters.com/bootcamp/)** — free intro path. Beyond it, FEM has the best video catalog (paid).

## Intermediate

- **[Patterns.dev](https://patterns.dev)** — design patterns for the modern web (rendering, performance, components).
- **[Total TypeScript (Matt Pocock)](https://www.totaltypescript.com)** — free tips + paid courses. The clearest TS material anywhere.
- **[Smashing Magazine](https://www.smashingmagazine.com)** — long-form articles, mixed quality. Filter by author.
- **[CSS-Tricks](https://css-tricks.com)** — older but the [Almanac](https://css-tricks.com/almanac/) and [snippets](https://css-tricks.com/snippets/) are evergreen.
- **[Kent C. Dodds — Epic Web](https://epicweb.dev)** — paid. Opinionated full-stack React (Remix-flavored).

## Books

- **Kyle Simpson — *You Don't Know JS Yet*** — free on GitHub. The clearest tour of the language's quirks.
- **Marijn Haverbeke — *Eloquent JavaScript*** — free online. Best end-to-end intro book; covers the language *and* the platform.
- **Lara Hogan — *Designing for Performance*** — short, opinionated, still relevant.
- **Heydon Pickering — *Inclusive Components*** — patterns for accessible component design.
- **Steve Krug — *Don't Make Me Think*** — UX fundamentals for engineers. ~3 hours to read; high ROI.

## Video — channels worth subscribing to

- **[Theo (t3.gg)](https://www.youtube.com/@t3dotgg)** — opinionated, plugged into the ecosystem. Skip the drama, keep the architecture takes.
- **[Web Dev Simplified](https://www.youtube.com/@WebDevSimplified)** — strong fundamentals series.
- **[Lee Robinson](https://www.youtube.com/@leerob)** — Vercel/Next.js deep dives.
- **[Fireship](https://www.youtube.com/@Fireship)** — 100-second concept summaries; useful for getting an overview, not for learning.
- **[Jack Herrington](https://www.youtube.com/@jherr)** — React, TypeScript, monorepos.
- **[Wes Bos / Syntax FM](https://syntax.fm)** — podcast. Solid for staying current passively.

## Backend / databases

- **[Hono docs](https://hono.dev/docs)** — small, complete framework docs. Read them in an evening.
- **[Drizzle ORM docs](https://orm.drizzle.team/docs/overview)** — the lightest TS-native ORM. Mental model is "SQL plus types."
- **[Use The Index, Luke!](https://use-the-index-luke.com)** — free book on SQL indexing. Essential once you have a real Postgres database.
- **[Postgres Weekly](https://postgresweekly.com)** — newsletter; weekly drip on Postgres news and tips.
- **[The Database Internals book (Alex Petrov)](https://www.databass.dev)** — only when you start caring *why* Postgres behaves the way it does.

## Performance / accessibility

- **[Vitaly Friedman / Smashing — Accessibility checklists](https://www.smashingmagazine.com/2021/03/complete-guide-accessibility-tools/)** — practical lists.
- **[a11y-collective](https://www.a11y-collective.com)** — short, focused lessons.
- **[Calibre's Web Performance handbook](https://calibreapp.com/handbook)** — free, deep, current.

## Communities

- **[r/webdev](https://www.reddit.com/r/webdev/)** — broad, mixed signal, useful for surveying.
- **[r/reactjs](https://www.reddit.com/r/reactjs/)** — narrower, higher quality.
- **[Hacker News](https://news.ycombinator.com)** — opinion-heavy; cross-check before adopting any "X is dead" take.

## Tools worth installing

- **[VS Code](https://code.visualstudio.com)** + extensions: ESLint, Prettier, EditorConfig, Error Lens, GitLens, the framework you use.
- **[React Developer Tools](https://react.dev/learn/react-developer-tools)** browser extension.
- **[axe DevTools](https://www.deque.com/axe/devtools/)** for a11y.
- **[Wave](https://wave.webaim.org)** for a different a11y perspective.
- **[Lighthouse](https://developer.chrome.com/docs/lighthouse/)** (built into Chrome dev tools).
- **[Bun](https://bun.sh)** — drop-in fast runtime/package manager. Try it on a side project.
- **[Biome](https://biomejs.dev)** — Prettier+ESLint replacement, much faster. Adopt once your team is ready.

## What to skip

- *Dev influencer Twitter takes* about which framework "won" or "died" this week. Pick one, ship things.
- *Tutorials older than ~3 years* for anything framework-shaped. The platform itself ages slower; old MDN articles are usually fine.
- *Boilerplates with 50 stars and a mile-long README.* Either use a recommended starter (Vite's templates, `create-next-app`) or roll your own from scratch.
- *Newsletter / course bundles you don't read.* Unsubscribe ruthlessly.

## Where to put your hours

A rough weighting that works for most people:

- 60% — *building and shipping projects.* Nothing else moves the needle as much.
- 15% — reading reference docs (MDN, framework docs) for things you're using *now*.
- 15% — focused video / book learning, paced to what your projects need next.
- 5% — reading other people's code (open source projects, finished personal sites).
- 5% — community / news / staying current.

If your weights look like the inverse (60% reading takes), you're in *learning mode* without a project. Pick a project from [Examples](./examples.md) and start.
